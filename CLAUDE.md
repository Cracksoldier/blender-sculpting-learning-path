# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the app

Open `index.html` directly in a browser — no build step, no server, no dependencies. Everything works from `file://`.

For a local dev server if needed: `python3 -m http.server 8080` then open `http://localhost:8080`.

## Architecture

Pure static app: no framework, no bundler, no external dependencies.

- **`index.html`** — shell only. All DOM structure for the curriculum and challenges is generated by `app.js` at runtime. The HTML just provides the sticky header skeleton (with progress bar slots), two `<section>` containers (`#levels-container`, `#challenges-container`), and loads the CSS/JS.
- **`app.js`** — single file containing both the data and all application logic.
  - `CURRICULUM` — nested array: levels → modules → topics. Each topic has `id`, `title`, `resourceUrl`, `resourceLabel`.
  - `CHALLENGES` — flat array of challenge objects with `id`, `title`, `difficulty`, `description`.
  - Progress state is a plain object `{ topics: Set, challenges: Set }` held in the module-level `progress` variable, persisted to `localStorage` under key `blender_sculpt_v1`.
  - `renderCurriculum()` / `renderChallenges()` fully rebuild their DOM sections from scratch. Exception: `renderSingleChallenge()` re-renders one card in-place via `replaceWith` to avoid re-attaching all listeners.
  - `topic._levelId` is set on the data object by `renderModule(mod, levelId)` before passing the topic to `renderTopic()`, so the change-event closure can call `refreshLevelHeader(levelId)` without needing to walk the DOM.
- **`style.css`** — uses CSS custom properties (`--lvl-1` through `--lvl-5` for the five level accent colours). Level colour variants are driven by `data-level` attribute selectors on `.level-block`. Challenge difficulty colours use `data-difficulty` attribute selectors on `.challenge-card`.
- **`favicon.svg`** — standalone SVG (wireframe sphere + brush), no external refs.

## Adding content

To add a topic, append to the relevant module's `topics` array in `CURRICULUM`. Each topic needs a unique `id` (convention: `l<level>-<module-abbrev>-<n>`).

To add a level, append to `CURRICULUM` and add the corresponding CSS variables and `data-level` selectors in `style.css` (border, badge background/colour).

## GitHub Pages deployment

Push the repo root to the `main` branch and enable Pages under repo Settings → Pages → Deploy from branch `main` / `/ (root)`. No build step required.
